Known Limitations

This section lists all known limitations of the MM.MySQL driver. Some of these items will get implemented some day, and others are due to shortcomings in the MySQL server.

JDBC-1.1 Methods not Implemented

The following methods from the JDBC-1.1 API are not implemented

  • Connection.prepareCall() - MySQL does not support stored procedures, so this method can not be implemented. Once MySQL supports some sort of stored procedure, this can be implemented.

  • DatabaseMetaData.getColumnPrivileges(...) - MySQL doesn't have a permissions system that maps onto SQL92, so this method is not implemented.

  • DatabaseMetaData.getCrossReference(...) - MySQL does not support foreign keys, so this method is not implemented.

  • DatabaseMetaData.getExportedKeys(...) - MySQL does not support foreign keys, so this method is not implemented.

  • DatabaseMetaData.getProcedureColumns(...) - MySQL does not support stored procedures, so this method is not implemented.

  • DatabaseMetaData.getProcedures(...) - MySQL does not support stored procedures, so this method is not implemented.

  • DatabaseMetaData.getSchemas() - MySQL does not have schemas, so this method is not implemented.
